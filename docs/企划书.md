# BookMirror 企划书

> 版本：内测阶段快照 · 与代码实况对齐 · 所有功能均已建成上线（Render 后端 + EAS 安卓包）

---

## 一、项目概述

BookMirror 是一款中文读书 App。它把三件事拧在一起：

1. **按「你这个人此刻的状态」推书**（MBTI × 星座 × AI 画像），而不是按销量或标签；
2. **AI 陪你读、接住你的情绪**（雪宝 / Miki 两个角色）；
3. **真正把书读完，并在段落里留下你的批注**（内测书库 + 阅读器 + 段落评论）。

技术上是 React Native/Expo 前端 + FastAPI/Python 后端，AI 用 Anthropic Claude。当前 zh 为主、en 已全量支持。

---

## 二、要解决的问题

- 推书要么按算法销量、要么按标签，很少「照见此刻的你」。
- 读书类 App 重工具、轻陪伴：没人接住「读到某一段时的情绪」。
- 想找一本只记得残缺片段（一个情节、一句话、一个画面）的书，搜索引擎无能为力。
- 读书是孤独的：书评在书外，读的过程里没有「和别人同处一段文字」的感觉。

---

## 三、目标用户

愿意做人格测试、愿意和 AI 聊内心、把读书当成自我照见的中文读者。手机端为主。

典型场景：分手/焦虑/迷茫时打开雪宝聊两句，被顺势推一本对的书；在书库里读《活着》，在某一段写下自己的批注，看到别人也在这一段留了话。

---

## 四、产品定位与差异化

一句话：**一面照见自己的镜子——用人格画像推书，用 AI 陪你读，用段落评论让读书不再孤独。**

差异化：
- 推荐维度是「人格 + 当下状态」（含从聊天里蒸馏出的画像），不是纯行为算法。
- AI 不是问答机器人，是有语气卡、有主见、会心理陪伴的角色（雪宝）。
- 阅读的核心特色是**段落级评论/笔记**（像微信读书的想法，但更进一步做成社交）。
- 有**凭残缺记忆找书**这种别处没有的入口。

## 五、功能详解（全部已建成）

### 5.1 人格测评与「星格」画像
- **MBTI**：53 题完整版 + 20 题快速版；也支持「我知道自己的 MBTI」直接填。AI（Claude tool_use）从答题推断 `{mbti, confidence, reasoning}`。
- **星座/星盘**：录入生日 → 太阳/月亮/上升、元素等（后端用 swisseph 真实星历计算）。
- **星格**：MBTI × 星座的综合人格画像 + LLM 生成的「塑造报告」（只做解读，不改 MBTI）。
- 展示在「我的账号」页：先星座后 MBTI，点头像进个人主页。

### 5.2 书籍推荐
- 141 本中文书库（另有 500 本扩展 catalog），每本打好标签（topic / problem / 难度 / MBTI 适配等）。
- 推荐链路：**SQL 风格候选筛选 → Claude 排序 + 生成「为什么适合你」→ 防幻觉校验**。会把从雪宝聊天蒸馏的画像也带进来，让推荐更贴近真实状态。
- 每日书单 + 全库搜索（中英书名/作者）。
- 书详情页：全屏、右滑入、可上/左/右滑出；含简介、为什么适合你、重点读什么、章节主题、关键词、去阅读平台的跳转、心形收藏、**在线阅读按钮**。

### 5.3 雪宝（小镜子）——AI 阅读与心理陪伴
这是产品的灵魂。
- **角色**：雪宝(Wren) 负责陪聊 + 荐书；Miki(小猫) 负责在后台总结用户画像与记忆。
- **语气卡**（严格约束，见 `services/mirror.py`）：像发微信、短句、少发问、陈述句收尾；禁破折号/「」/`*动作*`(改（）)/排比对比句/自我标榜；不叠甲不表演；抑郁双相直接接住、不甩锅给「去看专业」；语言纯净（英文模式全英文）。
- **专业底子**：内置 MI(动机式访谈)/ACT/CBT 疗愈工具箱，但用生活化语气，不自称心理咨询。
- **多模态**：图片/拍照/文件上传走 Claude vision；语音走端上识别（免费）。
- **grounded 荐书**：只能从后端给的真实候选书里选（tool 校验），杜绝编书。
- **多对话系统**：抽屉、新建对话、项目/文件夹、重命名/删除；消息按对话隔离，画像按用户全局。
- **消息体验**：气泡从小到大温柔弹出(PopIn)、长回复拆成多条泡泡渐次冒出、微信式时间线分隔；**长按消息弹治愈风行式菜单**：复制 / 引用 / 删除 / 转发(发给好友) / 收藏 / 多选。
- **增强记忆**：cares_about / mood_recent / reading_now / long_term，跨对话记住这个人。

### 5.4 读后评价与成长
- **三合一书评**：评分 + 难度 + 情绪词 + 心里话 + 是否推荐给相似人格 + 匿名 + 成长自评（表达/情绪/执行/自我认知/关系 五维，前后对比）。
- **阅读状态门控**：想读 / 在读(带进度) / 读完；读完才解锁评价（复用打卡系统，读完自动运算）。
- **Mirror Score**：推荐分公式。
- **塑造报告**：跨书聚合的成长曲线 + 这些书帮你解决了哪些问题（内耗/焦虑/拖延…）。

### 5.5 社交系统
- **用户 ID(handle)**：登录自动分配 9 位；可自定义、全局唯一、**区分大小写**(wren_test≠WREN_TEST)、**20 天最多改 3 次**；可被搜索。
- **找好友**：我的账号右上角 ＋ → 搜 ID 或用户名 → 进对方主页。
- **关注体系**：粉丝 / 关注 / 朋友(互关) / 访客，四栏可点进列表；看别人主页四栏换成 粉丝 / 关注 / ta的群聊(占位)。
- **备注**：看别人主页点 ＋ 起备注，备注覆盖显示名（主页/好友列表/聊天标题）。
- **隐私**：隐藏签名 / 资料 / 书评 / 收藏 / 访客足迹。
- **公开个人主页**：头像/用户名/@ID/签名/星座MBTI职业标签 + 四项计数 + 关注按钮 + 三栏(ta的书评 / ta的收藏 / ta的推荐)。
- **私信(DM)**：仅互关好友；QQ 式气泡弹出、消息按时间戳分组、**已读/未读回执**、**发图片**、长按复制/引用；小世界页顶「好友列表」栏显示会话+未读红点。
- **来信横幅**：好友发消息时从顶部弹跳横幅(头像+名+内容)，点击进聊天（App 开着时；轮询实现，非真推送）。

### 5.6 内测书库 + 阅读器（当前重点）
- **内测书库**：已录入正文、可真正在线读的书（区别于只做推荐的 catalog，无版权不可商用，仅内测）。
- **阅读器**：WebView + CSS 分栏做**仿纸书左右翻页**（浏览器原生排版，换字号自动重排，滚动顺）。
  - 设置实时生效：字号 / 行距 / 页边距 / 字体(黑体·宋体) / 背景(纸黄·护眼绿·夜间·纯白)，存本地。
  - 目录跳章；**进度按「第几章第几段」锚点存服务器**，换设备/重装续读不丢（与字号无关）。
- **段落评论（特色）**：长按段落或点段尾评论气泡 → 面板发**公开评论**(可点赞、别人能看) 或**私人笔记**(只自己看)。段尾显示评论数气泡。
- **入口**：推荐主页「内测书库」+ 每本书详情页「在线阅读」按钮（可读亮、不可读淡，按书名归一化匹配）。

### 5.7 AI 凭印象找书
- 输入残缺记忆片段 → 后端**检索全库全文 + 外部书目** → Claude 在候选里判断是哪本、给理由。
- 原理是「先检索出候选、再让 AI 认」（不是把全库塞给 AI），又快又准。
- 实测："一个老人和一头叫福贵的牛，讲一生苦难" → 认出《活着》并说明理由。

### 5.8 账号与多端同步
- 手机号验证码 / Google 登录。
- 账号档案（性别/星座/MBTI/用户名/职业/专业等）云端同步，换设备恢复。
- **收藏跟账号走**：本地 + 账号取并集对账，重登/换设备不丢。

## 六、技术架构

- **前端**：React Native + Expo SDK 54 + TypeScript；导航 native-stack；EAS 发布（build 出 APK、update 出 OTA，纯 JS 改动秒级下发）。
- **后端**：FastAPI + SQLAlchemy 2.0，部署 Render（Docker，push 自动部署，单实例）。
- **数据库**：SQLite（Render 持久盘）；用 SQLAlchemy 抽象，将来切 MySQL 只改 `DATABASE_URL`。共 **21 张表**（账号/小镜子/书评阅读/社交/阅读器五组）。
- **AI**：Anthropic Claude（推荐排序、MBTI 推断、雪宝对话、塑造报告、找书）。
- **推送**：直连 FCM HTTP v1（服务账号存 Render 环境变量，不经 Expo 中转）；仅安卓，iOS 待付费开发者账号。
- **内容管线**：本地脚本抽书 → POST 入库。文字 PDF(PyMuPDF)、扫描 PDF(RapidOCR)、Gutenberg 公版(繁→简 OpenCC)、EPUB(ebooklib)。服务器不装这些重库。

## 八、合规与版权立场（明确红线）

- **仅内测、无版权内容不可商用。**
- 书源**只用**：公有领域书（Gutenberg 等）、用户有权使用的文件、将来的正版授权。
- **明确拒绝**：抓第三方书站(wxbooks)、接盗版 API(Z-Library)、绕过对方反爬（Standard Ebooks 蜜罐）。
- 不硬编书封面/购买链接/书名/平台 id。
- 密钥（Claude Key、FCM 服务账号）绝不进 git。
