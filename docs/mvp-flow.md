# MVP 页面流程

第一版只做 4 个页面（结果页 + 推荐页合并）。语言放在设置里切换。不做账号，数据存本地。

---

## 页面 1：首页 `HomeScreen`

**目标**：30 秒让用户明白能干嘛，按下"开始"。

- Logo + slogan：「用一面镜子，照出你该读的下一本书」
- 主按钮：**开始测试**
- 次按钮：**查看上次推荐**（本地有记录才显示）
- 右上角齿轮 → 设置页（语言切换、清除数据、关于）

---

## 页面 2：测试页 `QuizScreen`

分 4 步 Stepper（手机长表单完成率极低）。

### Step 1 — MBTI 入口
- 按钮 A：「我知道自己的 MBTI」→ 4 组二选一（E/I, S/N, T/F, J/P）→ 拼成 MBTI
- 按钮 B：「我不知道自己的 MBTI」→ 选择测评方式：
  - **快速测评**：20 题（从 53 题中挑选最具代表性的题目）
  - **完整测评**：53 题
- 题型：是/否/看情况 三按钮 + 0–5 游标卡尺 + 自定义三选一
- 答题完成后调 `POST /api/mbti`，**Claude 推断 MBTI**，返回 `{ mbti, confidence, reasoning }`
- 输出：`{ mbti: "INFP", mbti_source: "self" | "quiz" }`

### Step 2 — 当前目标（多选，最多 3）
表达 / 情绪管理 / 职场 / 权谋 / 人际 / 学习方法 / 财商 / 恋爱关系 / 自律

### Step 3 — 阅读偏好（多选 + 滑块）
- 类型多选：小说 / 非虚构 / 历史 / 心理学 / 商业 / 哲学 / 传记
- 深度滑块：轻松 ←→ 深度（0–10）

### Step 4 — 当前问题（多选 + 自由输入）
预设：容易内耗 / 执行力差 / 不会拒绝别人 / 表达不清楚 / 太理想化 / 拖延 / 焦虑 / 想得多做得少
+ 选填 TextInput（≤100 字）

### 提交
- 显示骨架屏 + "Claude 正在为你画像中…"
- POST `/api/recommend`，流式接收

---

## 页面 3：结果 + 推荐合并页 `ResultScreen`

可滚动，从上到下：

1. **阅读画像卡片**：100 字描述 + 3 个关键词标签 + 分享按钮（占位）
2. **你现在最该补的能力**：3 条 bullet
3. **5 本书推荐**：卡片纵向列表，每张：封面 + 书名 + 作者 + 难度 + "为什么适合你"（2 句话引用用户具体输入）+ 折叠展开"重点读什么"（3 章节）。第 1 本标"⭐️ 从这本开始"。
4. **底部固定按钮**：「我读完了 → 反馈」 / 「重新测试」

---

## 页面 4：反馈页 `FeedbackScreen`

第一版只收集数据，不影响推荐。

- 选择书（下拉）
- 4 大按钮：✅ 有用 / 😵 太难 / 🥱 不感兴趣 / 🔁 想要类似的
- 选填 TextInput
- 提交 → 本地存 + POST `/api/feedback` → 回结果页

---

## 状态机

```
Home ──开始──> Quiz(1→2→3→4)
  ↑                  │
  │                  ▼
  └── Result ◄── /api/recommend (stream)
         │
         ├── 重新测试 → Quiz(1)
         └── 我读完了 → Feedback → Result
```
