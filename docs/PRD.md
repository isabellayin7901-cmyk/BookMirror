# BookMirror PRD

> 产品需求文档。改动产品范围前先改这里，再动代码。

## 一句话定位
一个「照见自己」的中文读书 App：用 MBTI×星座画像给你推书，用 AI（雪宝/Miki）陪你聊、帮你把书读进去，并在内测书库里真正把书读完、在段落里留下你的批注。

## 解决什么问题
- 市面推书要么按销量/算法，要么按标签，很少「按你这个人此刻的状态」推。
- 读书类 App 重工具、轻陪伴；没人接住「读到某一段的情绪」。
- 想找一本只记得残缺片段的书，搜索引擎无能为力。

## 目标用户
- 中文读者，愿意做人格测试、愿意跟 AI 聊内心的人。
- 手机端为主（Expo/React Native）。当前 zh 为主，en 已支持。

## 核心功能（已建成）
1. **推荐**：MBTI + 星座 + 小镜子画像 → 每日书单 / 搜索（500 本 catalog，只推荐、不可读，无版权）。
2. **雪宝（小镜子）**：AI 阅读与心理陪伴聊天。多对话/项目、图片/语音/文件输入、段落长按菜单（复制/引用/删除/转发/收藏/多选）、grounded 荐书（只从真实书库推，防幻觉）、Miki 做用户画像记忆。
3. **星格**：MBTI × 星座综合人格 + 塑造报告（LLM，不改 MBTI）。
4. **读后评价 + 成长**：三合一书评（评分/难度/情绪/文字/推荐/匿名/成长自评）、阅读状态门控、Mirror Score、塑造报告。
5. **社交**：ID(handle，20天改3次) + 找好友、关注/粉丝/朋友(互关)/访客、备注、隐私、公开主页、私信(已读回执/图片/时间线/气泡特效)、来信横幅。
6. **内测书库 + 阅读器**：已录入正文、可在线读的书（Gutenberg 公版 + 用户自有文件）。WebView 仿纸书翻页、字体/背景/边距设置、进度按锚点存服务器、**段落评论/笔记**（特色）。书详情页「在线阅读」按钮：可读亮、不可读淡。
7. **AI 凭印象找书**：残缺记忆片段 → 检索全库 → Claude 认书。

## MVP 边界 / 功能黑名单
- **不做盗版**：不抓第三方书站、不接盗版 API（Z-Library/wxbooks 已明确拒绝）。书源只用：公版（Gutenberg 等）、用户有权使用的文件、将来的正版授权。
- **不硬编**：不编造封面 URL / 购买链接 / 书名 / 平台 id。
- **群聊**：他人主页有「ta 的群聊」占位，功能未做。
- **iOS 推送**：需付费 Apple 账号，暂不做；推送当前仅安卓（FCM 直连）。
- **资治通鉴**：OCR 太大（8332页），未完成。

## 主要风险
- SQLite 单文件 + Render 单实例：并发写瓶颈、部署时 502 短暂。将来可迁 MySQL（DATABASE_URL 一行切换）。
- 推送安卓需 FCM 凭证（已配 google-services.json + Render 环境变量 FCM_SERVICE_ACCOUNT_JSON）。
- EAS 免费队列构建慢。
- 密钥：ANTHROPIC_API_KEY、FCM 服务账号绝不进 git。

## 验收标准（每个功能）
主流程能走通 + 加载中/为空/接口报错三态不崩 + zh/en 双语 + 不破坏既有数据。
